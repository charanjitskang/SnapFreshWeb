# SEO implementation notes

## What is implemented in code

- `robots.txt` and `sitemap.xml` are generated by `npm run generate:seo`.
- `npm run build` runs the SEO generation step automatically through `prebuild`.
- The public page list is centralized in `scripts/seo-config.mjs` so sitemap generation and canonical tags use the same route source of truth.
- The sitemap includes every current public page:
  - `/`
  - `/privacy/`
  - `/terms/`
  - `/support/`
  - `/data-deletion/`
- `robots.txt` allows crawling for all public pages and points crawlers to the sitemap URL.

## Site URL configuration

- The confirmed canonical production origin is `https://snapfresh.app`.
- Set `SITE_URL=https://snapfresh.app` in build and deploy environments so generated assets stay aligned with production.
- `VITE_SITE_URL` is also supported as a fallback input for SEO asset generation.
- If neither variable is set, the generator defaults to `https://snapfresh.app`.

## Intentional exclusions

- No public routes are intentionally excluded at this time.
- Internal API endpoints and build assets are not listed in the sitemap because they are not canonical public pages.

## Remaining operational steps

- Set `SITE_URL=https://snapfresh.app` in the deployment environment if it is not already present.
- Deploy the site and verify that `/robots.txt` and `/sitemap.xml` are live on the production origin.
- Confirm the canonical tag on each public page resolves to the matching absolute production URL after deploy.
- Complete Google Search Console verification using the method preferred by operations.
- Complete Bing Webmaster Tools verification.
- Submit `https://snapfresh.app/sitemap.xml` in both webmaster tools after deployment.
- Record the final verification method and ownership details once those steps are complete.
